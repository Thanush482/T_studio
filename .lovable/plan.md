# T_AI Studio — Phase 1 MVP Plan

Mobile-first PWA built on Lovable Cloud + Lovable AI Gateway. Focus on shipping a compliant, working creative app with subscriptions, then layer external models (Replicate/fal.ai) in Phase 2.

## Out of scope for this phase
- Video generation, video face swap, body swap, LoRA training, voice cloning, marketplace, white-label, API marketplace, government-ID (KYC) verification, dedicated GPU cluster. These need external providers or are Phase 3+.
- Real face-swap inference (no native model in Lovable AI). The UI will exist but be gated as "Coming soon — requires Replicate key" until you add one.

## Design direction
Dark cyberpunk. Background `#0a0a14`, surface `#13131f`, primary neon violet `#8b5cf6`, accent cyan `#06b6d4`. Font: Space Grotesk (display) + Inter (body). Tokens go in `src/styles.css` using `oklch`. Mobile-first layout with persistent bottom tab bar (Home / Create / Library / Profile).

## Database schema (Lovable Cloud)
- `profiles` — `user_id`, `display_name`, `avatar_url`, `tier` (free/pro/premium), `credits`, `verification_level` (email/phone/id), `created_at`
- `user_roles` — separate table, `(user_id, role)` with `app_role` enum (`user`, `moderator`, `admin`). Roles via `has_role()` security-definer fn (per the user-roles rule).
- `generations` — `id`, `user_id`, `tool` (t2i/edit/face_swap), `prompt`, `input_asset_paths[]`, `output_asset_path`, `model`, `status`, `moderation_result` (jsonb), `watermark_id`, `created_at`. Stores both visible-watermark output and invisible-watermark metadata (generation_id, timestamp, model, user_id).
- `consents` — `user_id`, `consent_type` (face_swap_subject_permission, training_ownership, tos, privacy), `version`, `ip_address`, `user_agent`, `created_at`. Append-only.
- `moderation_logs` — `id`, `generation_id`, `category`, `score`, `verdict`, `reviewer_id` (nullable), `created_at`.
- `abuse_reports` — `id`, `reporter_email`, `reported_generation_id` (nullable), `category` (deepfake/copyright/identity/harassment), `description`, `evidence_url`, `status`, `created_at`.
- `credit_ledger` — `id`, `user_id`, `delta`, `reason`, `generation_id` (nullable), `created_at`. Source of truth; `profiles.credits` is a denormalized cache updated by trigger.
- `subscriptions` — `user_id`, `tier`, `provider` (stripe), `external_id`, `status`, `current_period_end`.

All tables: explicit GRANTs + RLS. User-owned data scoped to `auth.uid()`. Admin/moderator overrides via `has_role()`.

## Authentication
- Email/password + Google (Lovable broker via `lovable.auth.signInWithOAuth`).
- `configure_social_auth` for Google.
- `onAuthStateChange` listener wired at root; profile auto-created via trigger on `auth.users` insert.
- Email confirmation required (no auto-confirm).

## Server functions (TanStack `createServerFn`, all `requireSupabaseAuth`)
- `moderatePrompt({prompt})` — Calls Gemini 2.5 Flash with strict policy prompt. Returns `{allow, categories, reason}`. Blocks: celebrity/politician names, minors, nude, fake-doc, fake-ID, violence, child terms. Logs to `moderation_logs`.
- `generateImage({prompt})` — Runs `moderatePrompt`, debits 1 credit (Free: 10/day cap), calls Lovable AI `openai/gpt-image-2` (or Nano Banana), saves to `generations` storage bucket, applies visible watermark "Generated by T_AI Studio", writes invisible metadata via PNG tEXt chunks, returns signed URL.
- `editImage({inputPath, prompt})` — Same flow with `google/gemini-3.1-flash-image-preview`.
- `faceSwapImage(...)` — Stub returning `{error: "provider_not_configured"}` until external key is added; documented for Phase 2.
- `recordConsent({type, version})` — Captures IP/UA via `getRequest()`.
- `reportAbuse({...})` — Public route (no auth) under `/api/public/report`.

## Subscriptions
- Lovable built-in Stripe payments (`enable_stripe_payments`).
- Products: **Pro ₹999/mo**, **Premium ₹1999/mo**, credit packs 100 / 500 / 1000.
- Free tier: 10 generations/day enforced server-side via `credit_ledger` query.
- Webhook handler at `/api/public/webhooks/stripe` updates `subscriptions` + tops up credits.

## Compliance backbone (non-negotiable, day-1)
- Consent modal before any face/edit feature; recorded in `consents`.
- Visible watermark on every output; invisible metadata in PNG.
- Hard-blocked prompts via moderation server fn (deny-list + LLM classifier).
- AI-generated label on all preview cards.
- Takedown form (public), abuse report form, response SLA copy on legal pages.
- Generations purge job: scheduled `pg_cron` daily — deletes assets + rows older than 30 days unless user pinned.
- Account deletion endpoint: cascades, hard-deletes within 90 days.
- Privacy / ToS / Acceptable Use / Takedown legal pages (templated, marked "review with counsel").

## PWA
- Manifest-only install (no service worker, per Lovable guidance) — icons, theme color, `display: standalone`, `start_url: "/"`.
